# Reproduction Details

This document records the full numerical results for the RaMA pipeline on the
M&Ms cardiac segmentation benchmark, reproduced from the open-source code in
this repository.

## Hardware

- Single NVIDIA RTX 5060 Ti / RTX 3090 (16–24 GB) is sufficient
- 1× Qwen-VL API access (DashScope-compatible) for LLM scoring

## Configuration that produced these numbers

### Source training (Step 2)

- **Optimizer**: SGD, momentum 0.99, nesterov, weight decay 5e-4
- **LR**: 1e-3 with `EpochLR` (gamma 0.9)
- **Loss**: Dice + BCE (`Seg_loss(['dice','bce'])`)
- **Epochs**: 400, batch size 8
- **Backbone**: Res-UNet with ResNet-34 (ImageNet pretrained)

### LLM scoring (Step 3a)

- **Model**: Qwen3-VL-Plus (DashScope-compatible)
- **Prompt**: per-organ single-organ overlay prompt (see `src/llm/system_prompts/`)
- **Thresholds**: discard < 40, refine 40–90, keep ≥ 90

### SAM refinement (Step 3b)

- **Agents**: SAM 3, MedSAM 2, SAM-Med2D
- **Fusion**: pixel-wise majority voting (≥ 2/3)
- **Fallback**: Dice<sub>vs pseudo</sub> < 0.5 → keep original pseudo-label
- **Mask weights**: LV 1.5, MYO 2.0, RV 2.0

### Round 0 (Step 4a)

- **LR**: 4e-5 with linear warmup 4 epochs + Cosine Annealing
- **Loss**: LLM-score-weighted Dice + curvature regularizer (β=0.01)
- **Epochs**: 50, batch size 8, freeze first 2 encoder layers, EMA decay 0.999

### Rounds 1–3 (Step 4b)

- **LR**: 3e-5 with warmup 2 epochs
- **Loss**: Dice + curvature (no LLM weighting)
- **Pseudo-label**: regenerated by the previous-round best model
- **Confidence filter**: per-pixel mean conf < 0.6 → discarded
- **ROI**: DBSCAN crop of the cardiac region
- **Epochs**: 30, batch size 8

## Data

- **M&Ms**: 4 vendors A/B/C/D, 1030/1342/525/550 train slices, 274/325/129/135 test slices
- **Pretrained ResNet-34**: 83 MB (torchvision auto-download or place at `weights/resnet34-333f7ec4.pth`)

For SAM agent code & weights see `README.md → External model code`.
