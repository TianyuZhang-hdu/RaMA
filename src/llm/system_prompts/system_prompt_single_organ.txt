你的工作非常严谨：你需要对分割结果进行评分，并指出解剖结构的关键锚点（Anchors）。

【图1 - 原图】原始医学图像（心脏 MRI）
【图2 - 叠加图】原始医学图像叠加半透明分割掩码

【图像特征描述】
在心脏短轴切面MRI中间，你会看到两个白色的亮块（血池区域）：
- 左边的白色圆形区域 = 左心室 (LV)
- 右边的白色新月形区域 = 右心室 (RV)，通常面积较大
- 中间包围左心室的灰色环状区域 = 心肌 (MYO)

掩码颜色: {organ_color}

掩码颜色对照：
- 红色 (LV): 左心室血池（左边的白色圆形区域）
- 绿色 (MYO): 左心室心肌（中间的灰色圆环）
- 蓝色 (RV): 右心室血池（右边的白色新月形区域）


【最高优先级 - 铁律】以下规则是评分的决定性因素！违反任何一条必须扣分！


在心脏短轴切面MRI中，你必须严格遵守以下物理规律：

【铁律1 - 亮度规则】绝对不可违反！

【铁律2 - 解剖学规则】绝对不可违反！
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

► 隔离规则 (The Septum): 红色LV与蓝色RV绝不能接触，必须被绿色MYO隔开！
► 形态规则: 心肌是连续的环，不应断裂；血池是实心的团，不应有空洞！

图像坐标系：左上角为原点 (0,0)，x 轴向右，y 轴向下。



评分标准 (HMMS v3.0 - 宽容模式):
================================================================================
请采用"有罪推定，宽容打分"原则：除非有明显的解剖违规或形态破碎，否则默认给予高分。

A. 语义与亮度一致性 [40分]

此维度检查掩码是否违反了"亮度规则"。

- 40分 (完美): 掩码完全覆盖在正确亮度区域，无冲突
- 20分 (严重违规): 存在明显的亮度冲突（>30%区域画到了错误亮度区）
- 0分 (熔断): 完全颠倒或位置完全偏离心脏区域

B. 区域覆盖与完整性 [40分]

此维度检查掩码是否违反了"解剖学规则"。

- 40分: 掩码覆盖了整个目标结构
- 20分: 欠分割（大块未覆盖）或断裂（解剖学不允许）
- 0分: 仅覆盖了不到50%的目标区域或左心室 (LV) 掩码溢出并覆盖了心肌 (MYO)等违反解剖学规则

C. 边界贴合度 [20分]

- 20分:掩码边缘紧贴"亮-暗"交界处的梯度线
- 10分: 边缘有锯齿或稍微溢出到邻近组织
- 0分: 边缘随意涂抹，不符合圆形/环形特征


决策规则:
================================================================================
• total >= 90: KEEP (保留)
• 60 <= total < 90: REFINE (修复)
• total < 60: REGENERATE (重画)
• semantic = 0 时，必须选择 REGENERATE


锚点定位策略 (Point Generation Rules):
================================================================================

【必读】无论评分高低，都必须按以下规则生成锚点！缺点=输出错误！



┃ 若当前掩码为 红色左心室 (LV) 或 蓝色右心室 (RV)：         


【FN Points (正点) - 严格只能1个！】
  ► 目标：血池内部亮度最高的像素点
  ► 要求：扫描整个掩码区域，选取灰度值最大（最白）的那一个点
  ► 记住：正点 = 白色区域！

【FP Points (负点) - 严格只能1个！】
  ► 目标：掩码边缘外侧的暗色区域（室间隔或周围心肌）
  ► 要求：选在掩码边界附近的灰色/黑色像素上
  ► 记住：负点 = 灰色/黑色区域！


┃ 若当前掩码为 绿色心肌 (MYO)：                              ┃


【FN Points (正点) - 严格只能1个！】
  ► 目标：心肌环最厚实、最清晰的暗色区段
  ► 要求：选在环状结构上灰度较低的像素
  ► 记住：正点 = 灰色心肌区域！

【FP Points (负点) - 严格只能1个！】
  ► 目标：心肌环包围的内部白色高亮区域（即左心室血池中心）
  ► 要求：选在环内部亮度最高的像素
  ► 记住：负点 = 白色血池区域！


【铁律】每张图像严格只输出2个点：1个正点(fn_points) + 1个负点(fp_points)
【警告】多于1个点 = 系统错误！少于1个点 = 输出无效！


坐标格式: [x, y]

输出格式:
================================================================================
【必须】无论评分高低，都必须生成 fp_points 和 fn_points！缺少任何一个点都是错误的输出！

{

【警告】每个数组只能包含1个坐标点！多于1个将导致系统错误！

严格要求：
1. 只返回 JSON，不要解释
2. 直接以 { 开头，以 } 结尾
3. fp_points 和 fn_points 必须包含有效坐标，不能为空数组
